# Support

PolyStella is maintained by a small team. We welcome reports,
questions, proposals, and pull requests, but cannot guarantee a prompt
response or review.

## Questions

Use GitHub Discussions for usage questions and design discussion:

https://github.com/cloudflare/polystella/discussions/categories/q-a

## Feature Requests

Use GitHub Discussions for feature proposals before opening a PR:

https://github.com/cloudflare/polystella/discussions/categories/ideas

Adapter, provider, cache, and routing changes should include the motivating use case because they often touch documented invariants.

## Bugs

Use the bug report issue form:

https://github.com/cloudflare/polystella/issues/new/choose

The most useful reports include a minimal `polystella.config.mjs` slice, a source fixture, provider details, and `dist/i18n-r2-report.json` when cache behavior is relevant.

## Security

Report vulnerabilities privately. See `SECURITY.md`.
