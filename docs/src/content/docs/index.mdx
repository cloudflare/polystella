  <CardGrid class="mt-14">
  <Card title="Build-time translation" icon="ph:rocket-launch">
    Translates `.md`, `.mdx`, and `.toml` content into additional locales during `astro build`. Visitors get static bytes; no runtime AI
    calls.
  </Card>
  <Card title="R2-cached" icon="ph:cloud-arrow-down">
    Translations are content-addressed by source bytes + glossary + model. Unchanged pages cost zero on rebuild. Translations are never
    committed to the repo.
  </Card>
  <Card title="Glossary control" icon="ph:book-open-text">
    A YAML file per locale pins do-not-translate terms, preferred translations, and free-form translator notes. Edits re-translate only the
    pages that need it.
  </Card>
  <Card title="Hand-translation overrides" icon="ph:pencil-simple-line">
    Drop a markdown file under `i18n/overrides/{locale}/...` and it wins over AI output verbatim. The pipeline still locale-prefixes its
    internal links.
  </Card>
  <Card title="Locale-prefixed routing" icon="ph:arrows-split">
    Per-locale sibling content collections. Astro routes `/[lang]/...` through generated shims. `Astro.locals.t` and `Astro.locals.lhref`
    work in templates.
  </Card>
  <Card title="UI-string maintenance" icon="ph:translate">
    Per-locale JSON files for chrome text. The CLI offers offline drift detection, key reconciliation, and AI-fill of empty placeholders.
  </Card>
  </CardGrid>
